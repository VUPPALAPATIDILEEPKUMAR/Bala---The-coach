# Chintu Bridge Contract V1

This document defines the shared file contract between the Windows build machine
and the iMac Control Room for the Chintu Bridge Sync system.

## Transport

Reports move from Windows to iMac via a shared bridge folder. The transport
mechanism (USB drive, cloud sync folder, network share, or manual copy) is
chosen by the founder. No automated network transfer.

## Direction

```text
Windows (build) --> BRIDGE/FROM_WINDOWS/ --> iMac (control room)
```

The iMac reads and archives. It does not write back to the repo. A future
`TO_WINDOWS/` folder is reserved for iMac-to-Windows instructions.

## File contract

Both machines use these exact filenames in `FROM_WINDOWS/`:

| Filename | Source | Content |
|---|---|---|
| `latest_status.md` | chintu-agent-board.ps1 | Overall Chintu status summary |
| `latest_bala_validation.md` | chintu-validate.ps1 | Last validation output |
| `latest_git_status.md` | git log | Recent git log and tree state |
| `latest_codex_handoff.md` | manual or agent board | Codex handoff or "parked" notice |
| `latest_openclaw_report.md` | chintu-openclaw-readiness.ps1 | OpenClaw readiness report |
| `latest_next_actions.md` | NEXT_SPRINT_QUEUE.md | Next sprint queue |

## iMac processing

When `bridge-sync.sh` runs on the iMac:

1. Read all `.md` files from `BRIDGE/FROM_WINDOWS/`.
2. Copy each to `REPORTS/YYYY-MM-DD/` (daily snapshot).
3. Copy each to `BRIDGE/LATEST/` (overwrite previous).
4. Move originals to `BRIDGE/ARCHIVE/YYYY-MM-DD/`.
5. Generate `BRIDGE/BRIDGE_STATUS.html` with sync timestamp and file list.
6. Log to `logs/bridge-sync.log`.

## Safety rules

- No secrets, tokens, or credentials in bridge files.
- No health data in bridge files.
- No executable content (scripts are separate from data files).
- Archive old reports; never delete.
- No automated network transfer.
- iMac scripts must be Catalina-safe bash.
- No installs required on either side.

## Folder structure

### Windows side

```text
C:\Users\Chintu\Desktop\test\
  (repo reports are gitignored; bridge-reporter.ps1 copies them out)
```

### iMac side

```text
~/Documents/CHINTU_CONTROL_ROOM/
  BRIDGE/
    FROM_WINDOWS/        <- drop zone
    TO_WINDOWS/          <- future
    LATEST/              <- most recent processed
    ARCHIVE/YYYY-MM-DD/  <- archived originals
    BRIDGE_README.md
    BRIDGE_STATUS.html   <- generated by bridge-sync.sh
  REPORTS/YYYY-MM-DD/    <- daily snapshots
  scripts/bridge-sync.sh
  logs/bridge-sync.log
```
