# Chintu Free Power Lanes

These are free, local, or open-source directions that may strengthen Chintu
later. Stage 10 documents them only. It installs, enables, downloads, connects,
or schedules nothing.

| Lane | Possible direction | Stage 10 status |
|---|---|---|
| Local LLM | Existing Ollama/local models; future local routing; Open WebUI-style concepts | parked |
| Local dashboard | Static HTML snapshot now; possible Electron/Tauri/PWA shell later | static snapshot only |
| Local speech | Browser speech APIs; Whisper-style transcription; Piper-style TTS | research parked |
| Local automation | Windows Task Scheduler, macOS launchd, Apple Shortcuts | activation parked |
| iMac/Windows control room | Windows build/validation/export plus iMac intake/archive | manual/local flow only |
| BALA safe product | Voice Coach, summaries, privacy/trust, tester polish | separate future sprints |

## Local LLM lane

Prefer local models for non-sensitive Chintu planning when reliability and
hardware cost are acceptable. Local routing must remain explicit, reviewable,
and unable to read secrets or health records by default. No model or dashboard
installation is part of Stage 10.

## Local dashboard lane

`CHINTU_AGENT_DASHBOARD.html` is the current zero-backend shell. Its generator
creates a factual snapshot from local reports. A desktop wrapper may be designed
later only if it improves usability without adding hidden network behavior.

## Local speech input/output research lane

Future research may compare browser SpeechRecognition/SpeechSynthesis with
local transcription and TTS tools. Voice cloning and real-person imitation are
prohibited. Nothing is installed, recorded, or generated in Stage 10.

## Local automation lane

Task Scheduler, launchd, and Apple Shortcuts may eventually run local report
refreshes. Scheduling remains parked because unattended execution changes the
operator and failure model. Cloud sync is never assumed or auto-configured.

## iMac/Windows control-room lane

Windows remains the brain/build/validation/export machine. The iMac remains the
control-room/intake/archive machine. iMac Option 12 must be tested before this
flow is described as smooth or reliable.

## BALA safe product lane

BALA work stays separate, local-first, non-medical, user-controlled, and behind
explicit founder selection. Chintu may prepare plans and test checklists but
does not move health data or change BALA during a Chintu OS infrastructure stage.

## Approval rule

Every lane stays parked until the founder selects it explicitly. Selection of
one lane does not activate any other lane, external channel, cloud behavior,
plugin, voice system, or background scheduler.
