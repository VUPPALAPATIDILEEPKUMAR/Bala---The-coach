# BALA Safe Copy Review

**Status:** planning. Read-only review notes. Operator-mode builder
sessions do not edit BALA app files. Founder-only to apply.

A checklist for reading every user-visible BALA string with the same
question in mind: *does this sentence make BALA sound like a
clinician, predictor, or emergency monitor?* If yes, soften it.

---

## 1. The replacement table

Use this whenever you spot a strong-claim word:

| Risky word / phrase | Safer wording |
|---|---|
| diagnose / diagnosis | signal / pattern / what your body might be telling you |
| treat / treatment | take care of / look after / give your body a break |
| predict / prediction | might be worth noticing / suggests / a heads-up |
| prevent / prevention | support / care for / build a habit around |
| emergency monitoring | daily check-in / awareness |
| replaces your doctor | a companion alongside your doctor |
| risk score | today's read / today's signal |
| alert / warning | notice / worth a look |
| cardiac arrest, heart attack | (never appears in app copy at all) |
| guaranteed / will improve | small steps can help / many people find |
| AI doctor / AI clinician | companion / coach / guide |

---

## 2. The preferred hero / sub-line

> "Your daily health guide from your own body signals."

Ownership ("your own"), softness ("guide"), no claim of accuracy. If
the existing hero is stronger than this, keep the existing one only
if it passes the §1 table; otherwise consider this replacement.

---

## 3. The preferred coach prefix

> "This is a companion, not a clinician."

One line. Visible on the first coach response of the session. Doesn't
have to repeat after that.

---

## 4. Safe verbs

When in doubt, prefer:

- notice
- listen
- check in
- balance
- care
- reflect
- support
- recover

Avoid:

- monitor (sounds clinical)
- track (sounds surveillance-y; ok in technical context, soft in
  user-facing)
- evaluate (sounds judgemental)
- assess (sounds clinical)
- detect (sounds diagnostic)

---

## 5. Numbers and scores

If a BALA score is shown:

- Frame it as today's *read*, not today's *risk*.
- Never compare across users.
- Never frame a low score as "danger" or "warning".
- Always pair the number with one short sentence of plain language.

---

## 6. Tone goals

Calm. Warm. Plain. Trustworthy. Like a friend who's reading the same
notes you wrote. Never alarmist. Never salesy. Never preachy.

Bad: *"Your sleep is dangerously low. Take immediate action!"*
Better: *"Sleep looks short the last few nights. Worth a check-in."*

---

## 7. Strings to re-read in person

Open these in the app and audit each one against §1:

- Home / hero line
- BALA Score card
- Today's coach card
- Each Health Signal Card
- Ask BALA Coach welcome
- Doctor-Ready Summary header (if shown)
- Import flow intro
- Settings / privacy page
- Footer disclaimer (must be visible, must include the safety footer)

---

## 8. What this review will never do

- Never produce diagnostic copy.
- Never produce predictive copy.
- Never produce emergency-monitoring copy.
- Never name a condition for the user.
- Never imply BALA replaces a clinician.

---

## 9. BALA safety footer

BALA is a health-awareness companion. It does not diagnose, treat,
predict, prevent, replace doctors, or provide emergency monitoring.
