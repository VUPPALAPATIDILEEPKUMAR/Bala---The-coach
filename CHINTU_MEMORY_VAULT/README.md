# Chintu Memory Vault

This is Chintu's local seed vault -- the foundational knowledge layer for the
BALA + Chintu multi-brain system.

## What this is

- A set of markdown files that capture product state, architecture, safety
  rules, release history, sprint queue, and founder preferences.
- Markdown-first and safe for human review at any time.
- Designed so any AI assistant (Claude, ChatGPT, Codex, or Chintu itself) can
  read these files and resume work with full context.

## What this is NOT

- The memory-wiki OpenClaw plugin is **not enabled**. These are plain files.
- No secrets, tokens, credentials, or private health data are stored here.
- No automated sync or external push. Everything stays local.

## Contents

| File | Purpose |
|---|---|
| `BALA_PRODUCT_STATE.md` | Current BALA features, stage, vision |
| `BALA_MEDICAL_SAFETY_RULES.md` | Prohibited claims, safe wording, boundaries |
| `CHINTU_AGENT_ARCHITECTURE.md` | Multi-brain roles and sprint flow |
| `RELEASE_HISTORY.md` | Recent important commits and milestones |
| `NEXT_SPRINT_QUEUE.md` | Ranked queue of upcoming work |
| `PARKED_SYSTEMS.md` | Systems intentionally disabled or deferred |
| `FOUNDER_PREFERENCES.md` | Builder preferences for collaboration style |
| `OPEN_QUESTIONS.md` | Decisions that need founder input |

## Rules

- Do not store health data, secrets, or credentials in this vault.
- Do not enable the memory-wiki plugin without explicit founder approval.
- Keep files accurate and up to date after each sprint.
- Any assistant may read; only commit changes after validation PASS.
