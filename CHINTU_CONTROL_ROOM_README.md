# Chintu Control Room

**Stage:** 10+
**Mode:** local-first, static, founder-driven

## What the control room is

The control room is Chintu OS's single entry point for the founder. It
connects every local report, dashboard, operator doc, validation script,
and builder handoff into one navigable index.

The control room is:

- `CHINTU_CONTROL_ROOM_INDEX.html`: generated static hub linking every file.
- `CHINTU_AGENT_DASHBOARD.html`: generated operator snapshot with system state.
- `scripts/chintu-control-room-index.ps1`: regenerates the index from repo state.
- `scripts/chintu-agent-dashboard.ps1`: regenerates the dashboard from reports.

## What it is not

It is not a live app, backend, or daemon. The HTML files are static
snapshots generated by local PowerShell scripts. They cannot read files
live, refresh automatically, or phone home. They never claim capabilities
that do not exist.

## How to use

Run the generators to refresh the snapshots:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\chintu-control-room-index.ps1
powershell -ExecutionPolicy Bypass -File scripts\chintu-agent-dashboard.ps1
powershell -ExecutionPolicy Bypass -File scripts\chintu-alive-briefing.ps1
powershell -ExecutionPolicy Bypass -File scripts\chintu-os-health-check.ps1
```

Then open `CHINTU_CONTROL_ROOM_INDEX.html` in any browser. It links to
every report and dashboard file in the repo.

## Refresh flow

1. Run existing operator scripts (daily operator, validators, bridge).
2. Run the new generators (alive briefing, health check, control room index).
3. Open the control room index in a browser.
4. Review facts, blockers, parked work, and the next exact action.
5. Founder decides on push, builder handoff, or next stage.

## Safety rules

- Local files only. No network, backend, or remote assets.
- No secrets, paid APIs, health-data transfer, or medical claims.
- No BALA app file edits during Chintu OS stages.
- No activation of parked systems without explicit founder approval.

BALA is a health-awareness companion. It does not diagnose, treat, predict, prevent, replace doctors, or provide emergency monitoring.
